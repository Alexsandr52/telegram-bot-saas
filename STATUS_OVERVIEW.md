## 🎯 Краткий обзор проекта

**Telegram Bot SaaS** - платформа для мастеров услуг, позволяющая создавать персональные Telegram-боты для записи клиентов.

### Основная идея
- Один бот мастера = один изолированный Docker-контейнер
- Простой интерфейс через Telegram
- Автоматическое создание и управление ботами

### Технологический стек
- **Backend:** Python 3.11+, aiogram 3.x, FastAPI
- **Database:** PostgreSQL 15+ с asyncpg
- **Cache/Queue:** Redis 7+
- **Containers:** Docker + Docker Compose
- **Frontend:** HTML/CSS/JS (Vanilla)

## ⚠️ Текущие проблемы (Критические)

### 1. Bot Template DB Connection 🔴
**Проблема:** Ошибки подключения к БД в bot-template
```
Error reloading config: connection is closed
```

**Причины:**
- DATABASE_URL может быть передан некорректно
- Контейнер не в сети `bot_saas_network`
- Проблемы в логике подключения

**Решение:** Проверить переменные окружения, сеть, добавить retry логику

### 2. Bot Status Mismatch 🟡
**Проблема:** `container_status` = "creating", но контейнер работает

**Решение:** Обновить статус в БД, добавить автоматическое обновление

### 3. Webhook Configuration 🟡
**Проблема:** `webhook_url` пустой в таблице `bots`

**Решение:** Настроить ngrok (dev) или публичный домен (prod)

## 🚀 Быстрый старт

```bash
# 1. Запустить сервисы
docker-compose up -d

# 2. Проверить статус
docker-compose ps

# 3. Просмотреть логи
docker-compose logs -f platform-bot

# 4. Настроить webhook (опционально)
./scripts/setup-ngrok.sh
python3 scripts/update-all-webhooks.py

# 5. Проверить webhook
python3 scripts/set-bot-webhook.py {TOKEN} get
```

## 📞 Поддержка

При возникновении проблем:
1. Проверить логи: `docker-compose logs -f {service_name}`
2. Проверить статус: `docker ps`
3. Проверить БД: `docker-compose exec database psql -U postgres -d bot_saas -c "SELECT 1"`
4. Проверить webhook: `python3 scripts/set-bot-webhook.py {TOKEN} get`
