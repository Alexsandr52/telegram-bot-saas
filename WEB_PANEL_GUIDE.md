## Запуск

### 1. Настройка переменных окружения

Добавьте в `.env` файл:

```bash
# Существующие переменные...
PLATFORM_BOT_TOKEN=your_platform_bot_token
ENCRYPTION_KEY=your_encryption_key
JWT_SECRET_KEY=your_jwt_secret_key

# Новые переменные
WEB_PANEL_URL=http://localhost:3000
CORS_ORIGINS=http://localhost:3000,http://localhost:80
```

### 2. Запуск через Docker Compose

```bash
# Запуск всех сервисов
docker-compose up -d

# Проверка статуса
docker-compose ps

# Логи
docker-compose logs -f platform-bot
docker-compose logs -f web-api
docker-compose logs -f web-panel
```

### 3. Доступ к сервисам

- **Веб-панель**: http://localhost:3000
- **Web API**: http://localhost:8000
- **API документация**: http://localhost:8000/docs
- **pgAdmin**: http://localhost:5050
- **Redis Commander**: http://localhost:8082

## Использование

### Шаг 1: Получение кода для входа

1. Откройте Platform Bot в Telegram
2. Нажмите **🌐 Веб-панель**
3. Скопируйте полученный код (например: `ABC12345`)

### Шаг 2: Вход в веб-панель

1. Откройте http://localhost:3000
2. Введите код из бота
3. Нажмите "Войти"

### Шаг 3: Управление

#### Просмотр ботов
- На главной странице отображаются все ваши боты
- Нажмите на бота для просмотра деталей

#### Записи
- Просмотр всех записей с фильтрацией по статусу
- Информация о клиенте и услуге
- Дата и время записи

#### Услуги
- Список всех услуг бота
- Цена и длительность
- Статус (активна/скрыта)

#### Расписание
- Рабочие часы для каждого дня недели
- Статус (работает/выходной)

## API Эндпоинты

### Авторизация
- `POST /api/v1/auth/login` - Вход с токеном
- `POST /api/v1/auth/logout` - Выход

### Боты
- `GET /api/v1/bots` - Список ботов мастера
- `GET /api/v1/bots/{bot_id}` - Детали бота

### Услуги
- `GET /api/v1/services/{bot_id}` - Список услуг
- `POST /api/v1/services/{bot_id}` - Создать услугу
- `PUT /api/v1/services/{service_id}` - Обновить услугу
- `DELETE /api/v1/services/{service_id}` - Удалить услугу

### Расписание
- `GET /api/v1/schedules/{bot_id}` - Получить расписание
- `PUT /api/v1/schedules/{bot_id}` - Обновить расписание

### Записи
- `GET /api/v1/appointments/{bot_id}` - Список записей
- `PUT /api/v1/appointments/{appointment_id}/status` - Обновить статус

## Troubleshooting

### Ошибка "Invalid or expired token"
- Токен действителен 24 часа
- Получите новый код в боте

### Бот не отвечает
- Проверьте логи: `docker-compose logs -f platform-bot`
- Убедитесь, что `PLATFORM_BOT_TOKEN` указан верно

### Веб-панель не доступна
- Проверьте, что контейнеры запущены: `docker-compose ps`
- Проверьте логи: `docker-compose logs -f web-panel web-api`

### База данных не подключается
- Убедитесь, что контейнер database запущен
- Проверьте строку подключения в `.env`

## Безопасность

- Токены действительны 24 часа
- Все API запросы требуют валидный токен
- CORS настроен для конкретных origins
- Пароли и токены зашифрованы в БД
