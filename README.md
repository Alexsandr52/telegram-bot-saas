Платформа для мастеров услуг, позволяющая создавать персональные Telegram-боты для записи клиентов.

## 📋 Обзор

**Telegram Bot SaaS** — это микросервисная платформа, которая позволяет мастерам услуг (парикмахеры, ногтевики, бровисты и т.д.) легко создавать и управлять Telegram-ботами для автоматического приема записей от клиентов.

### Основная идея
- **Один бот мастера = Один изолированный Docker-контейнер**
- Полностью автоматизированное создание и управление ботами
- Простой интерфейс через Telegram для мастеров и клиентов
- Веб-панель для расширенного управления

### Технологический стек
- **Backend:** Python 3.11+, aiogram 3.x, FastAPI
- **Database:** PostgreSQL 15+ с asyncpg
- **Cache/Queue:** Redis 7+
- **Containers:** Docker + Docker Compose
- **Frontend:** HTML/CSS/JavaScript (Vanilla)

### Тарифы
| Тариф | Цена | Лимиты | Функции |
|-------|------|--------|---------|
| **Free** | 0 ₽/мес | 1 бот, 50 записей | Базовая запись |
| **Pro** | 490 ₽/мес | 1 бот, безлимит | Напоминания, статистика |
| **Business** | 1990 ₽/мес | До 5 ботов | Команда, API-доступ |

## 🚀 Быстрый старт

### Предварительные требования
- Docker 24+
- Docker Compose Plugin
- Python 3.11+ (для локальной разработки)

### Локальный запуск

```bash
# 1. Клонировать репозиторий
git clone https://github.com/yourusername/telegram-bot-saas.git
cd telegram-bot-saas

# 2. Создать .env файл
cp .env.example .env
# Отредактировать переменные (токены, пароли)

# 3. Запустить все сервисы
docker-compose up -d

# 4. Проверить статус
docker-compose ps

# 5. Просмотреть логи
docker-compose logs -f platform-bot
```

### Использование Makefile

```bash
make help           # Показать все команды
make up             # Запустить все сервисы
make down           # Остановить все сервисы
make restart        # Перезапустить сервисы
make health         # Проверить здоровье сервисов
make logs           # Просмотреть логи
make db-backup      # Бэкап базы данных
```

## 🔧 Компоненты системы

### 1. Platform Bot (порт: 8001)
Главный бот для мастеров. Основные функции:
- Регистрация мастеров
- Подключение новых ботов через токен
- Управление услугами (CRUD)
- Настройка графика работы
- Просмотр записей
- Генерация кода для веб-панели
- Статистика мастеров

### 2. Factory Service (порт: 8002)
Управление Docker контейнерами ботов. Основные функции:
- Создание нового контейнера бота из шаблона
- Запуск/остановка/перезапуск контейнеров
- Удаление контейнеров
- Мониторинг статуса контейнеров
- Управление вебхуками

### 3. Bot Template
Шаблон бота мастера. Основные функции:
- Меню для клиентов
- Просмотр услуг
- Выбор даты и времени
- Запись на услугу
- Профиль клиента
- Генерация доступных слотов времени

### 4. Web API (порт: 8000)
REST API для веб-панели. Основные функции:
- Аутентификация мастеров
- CRUD операции для ботов
- CRUD операции для услуг
- Управление графиком
- Получение статистики
- Управление записями

### 5. Web Panel (порт: 3000)
Веб-интерфейс для мастеров. Основные функции:
- Вход через код из Telegram бота
- Дашборд со статистикой
- Управление ботами
- Управление услугами
- Настройка графика
- Просмотр записей

### 6. Notification Service
Система уведомлений и напоминаний. Основные функции:
- Напоминания клиентам за 24 часа
- Напоминания клиентам за 2 часа
- Уведомления о новых записях мастерам
- Уведомления об отмене записей
- Retry логика для неудачных отправок

## 📊 База данных

Основные таблицы:
- `masters` — Информация о мастерах
- `bots` — Информация о ботах мастеров
- `services` — Услуги мастера
- `schedules` — Рабочее время мастера
- `appointments` — Записи клиентов
- `clients` — Информация о клиентах
- `subscriptions` — Подписки мастеров
- `notifications_queue` — Очередь уведомлений

## 🔐 Безопасность

- Все токены ботов шифруются в базе данных
- Вебхуки защищены секретными токенами
- Используются параметризованные запросы для защиты от SQL Injection
- Rate limiting в nginx
