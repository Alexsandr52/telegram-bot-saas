## 🔧 Критические задачи (Blockers)

### 1. Исправление Bot Template DB Connection ⚠️
**Проблема:** Бот template имеет ошибки подключения к БД
```
Error reloading config: connection is closed
```

**Причины:**
- DATABASE_URL может быть передан некорректно
- Контейнер бота может быть не в сети `bot_saas_network`
- Некорректная обработка подключения в `bot-template/src/utils/db.py`

**Решение:**
- [ ] Проверить DATABASE_URL в environment переменных контейнера бота
- [ ] Убедиться что бот в сети `bot_saas_network`
- [ ] Проверить логику подключения в `bot-template/src/utils/db.py`
- [ ] Добавить retry логику для подключения к БД
- [ ] Добавить детальное логирование ошибок подключения

**Приоритет:** 🚨 КРИТИЧЕСКИЙ

## 📋 Важные задачи (High Priority)

### 4. End-to-End Тестирование
**Цель:** Протестировать полный flow системы

**Тест-кейсы:**
- [ ] **Flow 1: Создание бота мастером**
  - Мастер → Регистрация в Platform Bot
  - Мастер → Создание бота через @BotFather
  - Мастер → Подключение токена в Platform Bot
  - Factory Service → Создание контейнера
  - Бот → Успешный запуск
  - Бот → Ответ на /start

- [ ] **Flow 2: Настройка услуг**
  - Мастер → Добавление услуги через Platform Bot
  - Мастер → Настройка графика работы
  - Бот → Корректное отображение услуг
  - Бот → Генерация слотов времени

- [ ] **Flow 3: Запись клиента**
  - Клиент → /start в боте мастера
  - Клиент → Выбор услуги
  - Клиент → Выбор даты и времени
  - Бот → Создание записи в БД
  - Клиент → Подтверждение записи
  - Мастер → Уведомление о новой записи

- [ ] **Flow 4: Уведомления**
  - Notification Service → Создание напоминания за 24ч
  - Notification Service → Создание напоминания за 2ч
  - Клиент → Получение напоминаний
  - Retry логика при неудачной отправке

- [ ] **Flow 5: Отмена записи**
  - Клиент → Отмена записи
  - Бот → Обновление статуса в БД
  - Мастер → Уведомление об отмене
  - Очистка напоминаний

- [ ] **Flow 6: Web Panel**
  - Мастер → Вход через код из бота
  - Мастер → Просмотр дашборда
  - Мастер → Управление услугами через UI
  - Мастер → Настройка графика через UI
  - Мастер → Просмотр записей

**Приоритет:** 🔥 ВЫСОКИЙ

## 📊 План работ

### Неделя 1: Исправления и стабильность (16-24 часа)
1. **День 1-2:** Исправить Bot Template DB Connection (6-8 часов)
2. **День 3:** Обновить статусы ботов (2-4 часа)
3. **День 4:** Настроить webhook (локально через ngrok) (2-4 часа)
4. **День 5:** Базовое error handling (2-4 часа)

### Неделя 2: Тестирование (12-20 часов)
1. **День 1-2:** End-to-end тестирование Flow 1-2 (4-8 часов)
2. **День 3-4:** End-to-end тестирование Flow 3-6 (6-8 часов)
3. **День 5:** Фикс найденных багов (4-6 часов)

### Неделя 3: Документация и Production (12-20 часов)
1. **День 1-2:** Создание документации (4-8 часов)
2. **День 3-4:** Production настройка (6-8 часов)
3. **День 5:** Final testing и deploy (4-6 часов)

**Итого:** ~40-64 часов работы (~2-3 недели для одного разработчика)

## 📞 Поддержка разработки

При возникновении проблем:
1. Проверить логи: `docker-compose logs -f {service_name}`
2. Проверить статус контейнеров: `docker ps`
3. Проверить подключение к БД: `docker-compose exec database psql -U postgres -d bot_saas -c "SELECT 1"`
4. Проверить webhook: `python3 scripts/set-bot-webhook.py {TOKEN} get`
